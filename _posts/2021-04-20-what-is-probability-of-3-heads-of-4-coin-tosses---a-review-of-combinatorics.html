The basic formulation of combinatorics

Somewhat abstract and vague formulation

Combinatorics typically concerns with this problem: Given a countable set of possible states

`N = { a_1, .., a_n }`

and a set describing experiments

`X = { e_1, .. e_x }`

(either one experiment with `x` results or `x` experiments)

we can ask, how many possible results we can get when we run either one experiment with `x` results or `x` experiments with the states in `N`.

Formulation using balls and boxes

This formulation imagines that `N`, is a set of balls, which we can further imagine to be numbered, so

`N = { 1, .., n }`

and `X` is a set of boxes, to which we are placing the numbered balls.

We can ask about all the possible ways to place the balls into the boxes. Any ball can go into at most one box. However, we can distinguish the following experiment "types":

  • With/Without Replacement : After placing the ball into the box, we may allow to "replace" the ball we just used back into the set `N`, so it is available for the next experiment again, or keep it out of the set (making it not available).
  • With/Without Order: In addition, we can ignore the ball numbers ordering (or not).

The experiment "types" define 4 combinations. More on that later

Formulation using building a number from digits

Similar to balls and boxes, we can imagine we are building a number out of digits `1, .., n`. We do not restrict the number of digits in any way - we can use for example one Unicode point to one digit. But to make things simple, we simply imagine digits 0 to 9 or less.
